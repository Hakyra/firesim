Detailed Description

template<typename T>
class lbm::LBM< T >

Lattice Boltzmann Method fluid solver.

Uses the BGK collision model and Smagorinsky turbulence correction. The coordinate system used is left-handed, i.e. screen coordinates

The following boundary conditions are implemented:

  • no-slip Bounce back condition (fixed wall)
  • velocity Bounce back condition (moving wall)
  • inflow Inflow velocity with density of 1.0
  • outflow Enforces zero-gradient pressure at the boundary
  • pressure Enforces atmospheric pressure at the boundary

The constructor requires a configuration file in a ConfParser compatible format to be given, which must specify the following hierarchy of blocks:

  • domain Specifies the domain size
    • sizeX int Number of cells in x-dimension
    • sizeY int Number of cells in y-dimension
    • sizeZ int Number of cells in z-dimension
  • parameters Specifies the simulation parameters
    • omega float Inverse lattice velocity
    • cSmagorinsky float Smagorinsky turbulence constant
    • maxSteps int Number of time steps to simulate
  • vtk Specifies vtk output (optional, no output will be generated if this block is not present)
    • vtkFileName string Base name and path for the vtk file to write out (relative to location of executable, will be appended by current timestep and .vtk extension)
    • vtkStep int Number of timesteps in between 2 vtk file outputs
  • boundaries Specifies the boundary conditions, contains a subblock for each side of the domain (bottom, top, back, front, left, right) which in turn can contain 1 or more of the following subblocks to specify the type of boundary condition.
    • noslip Bounce back condition (fixed wall)
    • velocity Bounce back condition (moving wall)
      • u_x float Velocity of moving wall in x-direction
      • u_y float Velocity of moving wall in y-direction
      • u_z float Velocity of moving wall in y-direction
    • inflow Inflow velocity with density of 1.0
      • u_x float Inflow velocity in x-direction
      • u_y float Inflow velocity in y-direction
      • u_z float Inflow velocity in z-direction
    • outflow Enforces zero-gradient pressure at the boundary
    • pressure Enforces atmospheric pressure at the boundary
    Every boundary block needs to specify the coordinates of the rectangular patch, where the fixed coordinate is implicitly set and needs not to be specified (i.e. y for bottom and top, z for back and front and x for left and right.
    • xStart int Start cell in x-direction (not required for left and right)
    • xEnd int End cell in x-direction (not required for left and right)
    • yStart int Start cell in y-direction (not required for bottom and top)
    • yEnd int End cell in y-direction (not required for bottom and top)
    • zStart int Start cell in z-direction (not required for back and front)
    • zEnd int End cell in z-direction (not required for back and front)
    Boundary cells that are not specified are implicitly set to noslip. An error is thrown if a boundary cell is specified twice. bottom and top have priority over back and front which in turn have priority over left and right, i.e. the edges of the rectangle are added to the boundary regions according to these priorities given.

Example configuration file:

domain {
sizeX 62;
sizeY 62;
sizeZ 62;
}
parameters {
omega 1.9;
cSmagorinsky 0.03;
maxSteps 1201;
}
vtk {
vtkFileName vtk/outflow_60_60_60_1.9_0.03_1201_25;
vtkStep 25;
}
boundaries {
top {
outflow {
xStart 1;
xEnd 60;
zStart 1;
zEnd 60;
}
}
bottom {
inflow {
xStart 26;
xEnd 35;
zStart 26;
zEnd 35;
u_x 0.0;
u_y 0.0;
u_z 0.1;
}
}
left {
inflow {
zStart 2;
zEnd 59;
yStart 2;
yEnd 59;
u_x 0.001;
u_y 0.0;
u_z 0.0;
}
}
right {
outflow {
zStart 2;
zEnd 59;
yStart 2;
yEnd 59;
}
}
back {
outflow {
xStart 1;
xEnd 60;
yStart 2;
yEnd 59;
}
}
front {
outflow {
xStart 1;
xEnd 60;
yStart 2;
yEnd 59;
}
}
}
Note
define the preprocessor symbol NSMAGO to disable turbulence correction
